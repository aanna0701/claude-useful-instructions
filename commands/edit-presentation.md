description: 이미 포맷된 HTML PPT의 내용을 지시사항에 따라 수정한다. CSS/JS/템플릿 구조는 절대 변경하지 않는다.
---

# edit-presentation

기존 포맷 HTML PPT를 읽고, 지시사항에 해당하는 슬라이드 내용만 정확히 수정한다.

---

## 입력

`$ARGUMENTS`: `[input.html] [수정 지시사항]`

인자가 없으면 아래를 요청한다:

> "수정할 HTML 파일 경로와 변경 지시사항을 알려주세요.
> (예: '3번 슬라이드 제목을 X로 바꿔줘', '5번 카드 내용 추가', '날짜를 2026.04.01로 변경')"

---

## 절대 규칙 (위반 금지)

1. **CSS `<style>` 블록 — 한 글자도 수정하지 않는다**
2. **`<script>` 블록 — 한 글자도 수정하지 않는다**
3. **`.slide-header` 구조 — 수정하지 않는다** (로고 경로, headerLabel 포함)
4. **`.slide-nav` 구조 — 수정하지 않는다**
5. **지시사항에 언급되지 않은 슬라이드 — 절대 건드리지 않는다**
6. **클래스명, data-slide 번호, `.slide-label` — 변경하지 않는다**
7. **컴포넌트 구조(card-grid, flow-box, timeline 등) — 지시사항에서 명시하지 않는 한 유지**

---

## 실행 단계

### Step 1: 파일 읽기

입력 HTML 파일을 Read한다.

파일이 표준 포맷(`.slide-deck`, `.slide.slide-dark`, `.slide-header`)을 따르고 있는지 확인.
포맷이 맞지 않으면 사용자에게 알린다:
> "이 파일은 표준 포맷이 아닙니다. /format-presentation으로 먼저 변환해주세요."

### Step 2: 지시사항 파싱

수정 지시사항을 분석하여 아래 표로 정리한다 (수정 시작 전 사용자에게 보여줌):

| 슬라이드 | 수정 대상 | 변경 전 | 변경 후 |
| -------- | --------- | ------- | ------- |
| N번      | 제목 텍스트 | "..." | "..." |
| N번      | 카드 3번 내용 | "..." | "..." |

수정 범위가 불명확하면 먼저 확인한다:
> "3번 슬라이드의 어떤 카드를 수정할까요? (1/2/3번째)"

### Step 3: 내용 수정

**Edit 툴로 해당 위치만 정확히 교체한다.**

수정 원칙:
- 텍스트만 교체 — 감싸는 HTML 태그/클래스는 유지
- 이모지 변경이 지시된 경우만 이모지 교체
- 슬라이드 추가/삭제는 명시된 경우에만, `data-slide` 번호 재정렬 필수
- highlight 색상 변경(`<span class="highlight">`)은 명시된 경우만

### Step 4: 검증 및 출력

수정 후 아래를 확인:

- [ ] CSS/JS 블록 변경 없음
- [ ] 수정되지 않은 슬라이드 내용 그대로
- [ ] `.slide-label` 텍스트가 헤더 라벨과 일치
- [ ] `data-slide` 번호 연속성 유지

수정 완료 후 변경 요약 출력:

```
수정 완료: {파일명}
변경된 슬라이드: 3, 7번
총 변경 항목: 4개
```

덮어쓰기 대신 새 파일로 저장하려면 `-o output.html` 인자를 추가한다.
기본값: 원본 파일 덮어쓰기 (Edit 툴 사용).

---

## 예시 지시사항

```
# 단순 텍스트 변경
"1번 슬라이드 날짜를 2026.04.01로 바꿔줘"
"3번 슬라이드 제목에서 'Manual'을 'Complex'로 변경"

# 내용 추가/수정
"5번 슬라이드 카드 3개에 각각 내용 한 줄씩 추가해줘: ..."
"9번 표에 행 하나 추가 — 컴포넌트: Evaluation, 상태: In Progress"

# 슬라이드 추가
"7번과 8번 사이에 새 슬라이드 추가 — 제목: ..., 카드 3개: ..."

# 슬라이드 삭제
"11번 슬라이드 삭제"

# 복합 수정
"1번 발표자를 'Leo'에서 'Jay'로, 날짜를 2026.04.15로 변경하고
 6번 슬라이드 2행 카드들의 제목을 모두 굵게 처리해줘"
```
