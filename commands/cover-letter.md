# cover-letter — 자소서 작성 멀티 에이전트 시스템

NotebookLM MCP 기반 한국어 자기소개서(자소서) 작성 시스템.
Writer 에이전트와 Reviewer 에이전트가 반복 개선 루프를 돌며 고품질 자소서를 완성한다.

---

## Prerequisites

- **NotebookLM MCP** 서버가 연결 및 설정되어 있어야 한다
  - MCP Server: `https://github.com/jacob-bd/notebooklm-mcp-cli`
  - **"자소서"** 이름의 노트북에 이력서/포트폴리오가 저장되어 있어야 한다

---

## Step 1: 사용자 입력 수집

다음 정보를 사용자에게 확인한다:

1. **자소서 항목 (Question/Prompt)**: 지원서의 정확한 질문/주제
2. **강조 사항 (Emphasis Points)**: 강조하고 싶은 경험, 기술, 프로젝트, 특성
3. **글자수 제한 (Character Limit)**: 줄바꿈 포함 HARD limit. 없으면 500-800자 기본값 사용
4. **지원 회사/직무 (Company/Position)**: 대상 회사와 역할 (있으면)

---

## Step 2: NotebookLM에서 컨텍스트 조회

NotebookLM MCP 도구를 사용하여 "자소서" 노트북을 검색한다.

**쿼리 전략:**
- 먼저 사용자 배경의 전체 요약 조회
- 그 다음 특정 강조 사항과 관련된 타겟 쿼리 실행
- 구체적인 경험, 숫자, 프로젝트명, 성과를 추출

조회한 컨텍스트를 저장 — Writer와 Reviewer 모두 필요.

---

## Step 3: Writer 에이전트 — 자소서 초안 작성

**cover-letter-writer** 에이전트에게 위임한다.

전달할 정보:
- 자소서 항목
- 강조 사항
- NotebookLM 컨텍스트
- 글자수 제한
- (2회차 이후) Reviewer 피드백

Writer 에이전트의 핵심 역할:
- NotebookLM 컨텍스트로 **한글** 자소서 초안 작성
- 자소서 항목에 직접 대응
- 강조 사항을 자연스럽게 녹여냄
- 글자수 제한 엄수 (줄바꿈 포함 카운트)
- AI가 아닌 사람이 쓴 것 같은 자연스러운 톤

---

## Step 4: Reviewer 에이전트 — 초안 평가

**cover-letter-reviewer** 에이전트에게 위임한다.

전달할 정보:
- 자소서 항목
- 강조 사항
- Writer의 초안
- NotebookLM 컨텍스트
- 글자수 제한
- (2회차 이후) 이전 피드백

Reviewer 에이전트의 7가지 평가 항목:
1. 문법/맞춤법
2. 자연스러움
3. 사실 검증 (NotebookLM 대조)
4. AI 스타일 탈피
5. 항목 적합성
6. 구성/구조
7. 글자수 준수

각 항목 25/50/75/100점 → 총점 = 7개 평균.

---

## Step 5: 반복 개선 루프

```
WHILE total_score < 76 AND iteration < 5:
    IF iteration > 1:
        Writer가 Reviewer 피드백 기반으로 재작성
    Reviewer가 새 초안 평가 (각 항목 25/50/75/100점)
    total_score = 7개 항목 점수 평균
    개선 기록에 이터레이션 로그
    iteration += 1
```

**최대 5회 반복**. 5회 후에도 76점 미만이면 최고 점수 버전을 남은 피드백과 함께 제시.

---

## Step 6: 개선 기록 생성

루프 완료 후 전체 과정을 문서화한 Markdown 파일을 생성한다.

파일명: `자소서_개선기록_[항목요약]_[timestamp].md`

**기록 형식:**
```markdown
# 자소서 개선 기록

## 기본 정보
- 자소서 항목: [항목]
- 강조 사항: [강조 사항]
- 글자수 제한: [제한] (없으면 "없음")
- 최종 점수: [총점]/100 ([등급])
- 총 반복 횟수: [N]회

---

## 1차 작성

### Writer 작성 결과
[자소서 전문]

### Reviewer 평가

| 평가 항목 | 점수 | 등급 |
|-----------|------|------|
| 문법/맞춤법 | [25/50/75/100] | [등급] |
| 자연스러움 | [25/50/75/100] | [등급] |
| 사실 검증 | [25/50/75/100] | [등급] |
| AI 스타일 탈피 | [25/50/75/100] | [등급] |
| 항목 적합성 | [25/50/75/100] | [등급] |
| 구성/구조 | [25/50/75/100] | [등급] |
| 글자수 준수 | [25/50/75/100] | [등급] ([현재]자/[제한]자) |
| **총점** | **[평균]/100** | **[종합 등급]** |

#### 세부 피드백
[각 항목별 피드백]

---

## 2차 작성
(동일 구조 반복)

---

## 최종 자소서
[최종 자소서 전문]
```

---

## Step 7: 결과 제시

사용자에게 다음을 전달한다:
1. **최종 자소서** 전문
2. **개선 기록** `.md` 파일
3. 프로세스 요약 (반복 횟수, 최종 점수, 주요 개선 사항)

---

## 핵심 규칙

### 언어 규칙
- 자소서 출력은 반드시 **한국어(한글)** — 예외 없음
- Reviewer 피드백은 한국어로 작성
- 개선 기록 내 자소서와 피드백은 한국어

### 글자수 엄수
글자수 제한이 있을 때:
- 한글, 공백, 구두점, 줄바꿈 모두 카운트
- 줄바꿈(`\n`) = 1자
- Writer가 작성 후 셀프 체크
- Reviewer가 검증 후 위반 시 지적
- HARD limit — 1자라도 초과하면 불가

### AI 스타일 탈피
Writer가 피해야 할 것:
- 매 단락 "저는"으로 시작
- "다양한 경험을 통해", "이를 통해 성장할 수 있었습니다" 남발
- 구체적 예시 없는 추상적/모호한 진술
- 지나치게 매끄러운 로봇 같은 문장 구조
- "귀사에서 성장하고 싶습니다" 같은 일반적 결론

Writer가 추구해야 할 것:
- 구체적 일화와 수치/지표
- 다양한 문장 시작과 길이
- 자연스러운 대화체 흐름 (전문적이되 이야기하듯)
- 개성과 진솔한 성찰
- THIS 회사/역할을 원하는 구체적 이유

### NotebookLM 사용
- 사실 검증을 위해 반드시 "자소서" 노트북 조회
- 노트북에 없는 경험, 프로젝트, 성과를 절대 지어내지 않는다
- 노트북에 정보가 부족하면 지어내지 말고 사용자에게 질문
