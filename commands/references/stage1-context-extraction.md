# Stage 1: Context Extraction

NLM을 **추론 엔진**으로 활용. 단순 추출이 아니라 교차 분석 합성 질문.

## Query Strategy

| ❌ 추출 (NLM 낭비) | ✅ 합성 (NLM 활용) |
|---|---|
| "이력서 요약해줘" | "CV와 포트폴리오 비교해서 핵심 강점 패턴 분석해줘" |
| "기술 스택 목록" | "가장 깊은 전문성 분야를 근거와 함께 판단해줘" |

### Round 1 — 교차 분석
```
"전체 경력에서 반복적 핵심 역량 패턴 3가지를 문서 근거와 함께 정리해줘"
"CV에는 없지만 포트폴리오에서 드러나는 숨겨진 강점이 있어?"
"기술적 전문성이 가장 깊은 분야는? 논문과 프로젝트 기술서 기반으로 판단해줘"
"모든 프로젝트에서 역할의 공통점과 성장 궤적을 분석해줘"
```

### Round 2 — 구조화 추출 (합성 포함)
```
"모든 문서 종합해서 기술 스택을 카테고리별로 정리해줘"
"각 프로젝트별: 이름, 기간, 팀규모, 역할, 기술, 핵심과제, 수치 성과 — 여러 문서 합쳐서"
"경력 타임라인 연도순: 회사, 직책, 기간, 핵심 기여"
```

### Round 3 — Gap 분석
```
"수치나 성과가 빠진 프로젝트가 있어?"
"문서들 간 불일치하는 부분 (기간, 역할명, 수치)?"
```

## Output Format
```markdown
# 자소서 컨텍스트 정리
## 1. 기술 스택 (카테고리별)
## 2. 프로젝트 경험 (프로젝트별: 기간/팀/역할/기술/과제/성과)
## 3. 직무 성과 (수치화)
## 4. 경력 타임라인 (연도순)
## 5. 강점 포인트 (기술적/리더십/문제해결/도메인)
## 6. NLM 분석 인사이트 (패턴, 숨겨진 강점, 성장 궤적)
## 7. Gap & 불일치 ([NLM: 근거 부족] 표기)
```

## Upload
사용자 확인 후: `nlm source add "자소서" --text "..." --title "컨텍스트_정리_YYYYMMDD_HHMM"`
