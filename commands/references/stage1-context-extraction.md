# Stage 1: Context Extraction

> **Reuse Guard**: 실행 전 `nlm source list "자소서"`로 `컨텍스트_정리_*` 소스 존재 여부 확인.
> 존재하면 재사용 제안 → 사용자가 '재생성' 입력 시에만 아래 진행.

NLM을 **추론 엔진**으로 활용. 단순 추출이 아니라 교차 분석 합성 질문.

## Query Strategy

| ❌ 추출 (NLM 낭비) | ✅ 합성 (NLM 활용) |
|---|---|
| "이력서 요약해줘" | "CV와 포트폴리오 비교해서 핵심 강점 패턴 분석해줘" |
| "기술 스택 목록" | "가장 깊은 전문성 분야를 근거와 함께 판단해줘" |

### Round 1 — 교차 분석
```
"전체 경력에서 반복적 핵심 역량 패턴 3가지를 문서 근거와 함께 정리해줘"
"CV에는 없지만 포트폴리오에서 드러나는 숨겨진 강점이 있어?"
"기술적 전문성이 가장 깊은 분야는? 논문과 프로젝트 기술서 기반으로 판단해줘"
"모든 프로젝트에서 역할의 공통점과 성장 궤적을 분석해줘"
```

### Round 2 — 구조화 추출 (합성 포함)
```
"모든 문서 종합해서 기술 스택을 카테고리별로 정리해줘"
"각 프로젝트별: 이름, 기간, 팀규모, 역할, 기술, 핵심과제, 수치 성과 — 여러 문서 합쳐서"
"경력 타임라인 연도순: 회사, 직책, 기간, 핵심 기여"
```

### Round 3 — Gap 분석
```
"수치나 성과가 빠진 프로젝트가 있어?"
"문서들 간 불일치하는 부분 (기간, 역할명, 수치)?"
```

## NLM 결과 비판적 처리 (필수 — 출력 전 수행)

NLM 응답은 **원자료(raw data)**다. 그대로 재포매팅하지 말고, 수집 완료 후 아래를 AI가 독립적으로 판단한다.

**판단 항목:**
1. **서사 잠재력** — 어떤 경험/프로젝트가 가장 강한 스토리를 만들 수 있는가? NLM은 중요도를 모른다.
2. **시장 희소성** — NLM이 나열한 역량 중 실제로 희귀한 것은 무엇인가? 흔한 것은 평가 절하.
3. **NLM 압축 손실** — 합성 과정에서 사라진 뉘앙스나 맥락이 있는가? (수치 없는 성과, 팀 규모 누락 등)
4. **모순/과장 감지** — NLM이 문서 간 불일치를 무시하고 그럴듯하게 합성했는지 검증.
5. **Stage 3 예측** — 이 컨텍스트가 어떤 JD 유형에 가장 강하게 쓰일 수 있는가? 미리 태깅.

**규칙:** 판단 결과는 출력의 각 섹션에 `[AI 판단: ...]` 형태로 인라인 표기.

---

## Output Format
```markdown
# 자소서 컨텍스트 정리
## 1. 기술 스택 (카테고리별)
## 2. 프로젝트 경험 (프로젝트별: 기간/팀/역할/기술/과제/성과)
## 3. 직무 성과 (수치화)
## 4. 경력 타임라인 (연도순)
## 5. 강점 포인트 (기술적/리더십/문제해결/도메인)
   - [AI 판단: 희소성 평가 + 서사 잠재력]
## 6. NLM 분석 인사이트 → AI 재해석
   - NLM 원문: "..."
   - AI 판단: (동의/보정/반박 + 근거)
## 7. Gap & 불일치 ([NLM: 근거 부족] / [AI: 압축 손실 의심] 구분 표기)
```

## Upload
사용자 확인 후: `nlm source add "자소서" --text "..." --title "컨텍스트_정리_YYYYMMDD_HHMM"`
