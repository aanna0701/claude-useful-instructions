# create-presentation

주어진 내용을 분석해서 구조화된 HTML 슬라이드 덱을 생성한다.
포맷은 자유롭게 선택하되, 시각적으로 깔끔하고 읽기 쉬운 결과물을 만든다.

---

## 입력

`$ARGUMENTS`: 파일 경로 또는 텍스트

인자가 없으면 요청:
> "발표할 내용을 붙여넣거나 파일 경로를 알려주세요."

---

## 실행 단계

### Step 1: 내용 파악

파일 경로면 Read, 텍스트면 바로 분석.

아래를 추출:
- 핵심 주제와 목적
- 논리 흐름 (문제 → 해결 → 결과 등)
- 데이터/수치가 있는지
- 발표 대상 (기술팀? 경영진? 일반?)

### Step 2: 슬라이드 구조 설계

내용을 슬라이드 단위로 분해. 설계 원칙:

- 슬라이드당 **하나의 메시지**
- 텍스트 덩어리는 시각 요소(카드, 표, 다이어그램)로 변환
- 흐름: 표지 → 문제/배경 → 본론 → 결론/요약
- 권장 슬라이드 수: 6~12장

### Step 3: HTML 생성

아래 기준으로 자유롭게 디자인:

**레이아웃 선택 기준**
- 3가지 비교 → 3열 카드
- 단계/순서 → 화살표 플로우 또는 타임라인
- 숫자 강조 → 큰 stat 블록
- 목록 → 아이콘 카드 (단순 `<ul>` 지양)
- 표 → styled table (zebra stripe 등)

**디자인 기준**
- 폰트: Inter 또는 시스템 sans-serif
- 16:9 비율
- 다크/라이트 자유 선택 (내용 분위기에 맞게)
- 키보드 네비게이션 (←→ 키, 스페이스바)
- 하단 슬라이드 번호 표시

**금지사항**
- 슬라이드당 텍스트 7줄 초과
- 작은 글씨로 내용 욱여넣기
- 의미 없는 장식 요소

### Step 4: 저장

파일명: `{주제}-presentation.html` (현재 디렉터리)

저장 후 슬라이드 목록 요약:
```
생성 완료: topic-presentation.html
슬라이드 수: N장

1. 표지 — [제목]
2. [섹션명] — [레이아웃 타입]
...
```

---

## 포맷 업그레이드

생성 후 표준 포맷으로 통일하려면:
```
/format-presentation topic-presentation.html logo.png
```
