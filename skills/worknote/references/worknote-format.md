# Local Worknote Format

## Directory Structure

```
~/.claude/worknote/
└── YYYY-MM-DD/              # One directory per day
    ├── <repo-name>.md       # One file per repo (auto-generated by hook)
    └── <repo-name>.md
```

Each file is **overwritten** on every session end — always contains the latest snapshot for that repo on that day. No duplicates.

## File Format

```markdown
## <repo-name>

_Last updated: HH:MM_

### Commits
- <hash> <commit message>
- <hash> <commit message>

### Staged
 <file> | <stat>

### Changed
 <file> | <stat>
```

### Sections

| Section | Source | Present when |
|---------|--------|-------------|
| `Commits` | `git log --since=06:00` | At least one commit today |
| `Staged` | `git diff --cached --stat` | Staged but uncommitted changes |
| `Changed` | `git diff --stat` | Unstaged modifications |

All sections are optional — only included when data exists.

## Reading Worknotes

- **Single repo today**: `~/.claude/worknote/YYYY-MM-DD/<repo>.md`
- **All repos today**: `~/.claude/worknote/YYYY-MM-DD/*.md`
- **Date range**: iterate `~/.claude/worknote/YYYY-MM-DD/` directories
- **Specific project across dates**: glob `~/.claude/worknote/*/<repo>.md`

## Retention

- **Local**: Directories older than 30 days are auto-deleted by the `worknote-stop` hook on each session end.
- **Notion**: Pages older than 30 days are set to `Status: archived` by the `worknote-sync` agent during sync. Never deleted.

## Relationship to Notion

Local files are the source of truth. The `worknote-sync` agent reads per-repo files and creates/updates one Notion page per `(Date, Project)` pair. See `notion-schema.md` for DB schema.
