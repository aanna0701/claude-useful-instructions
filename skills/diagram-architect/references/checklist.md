# Diagram Checklist — 검증 체크리스트

Phase 4에서 모든 다이어그램에 대해 실행하는 검증 항목.

---

## 필수 체크 (하나라도 실패 → 수정 필수)

### 단순성
- [ ] 도형 15개 이하인가? (초과 → 분할)
- [ ] 도형 내 텍스트 5단어 이하인가?
- [ ] subgraph 중첩 1단계 이하인가?

### 방향성
- [ ] 주 흐름이 한 방향(LR 또는 TB)으로 일관된가?
- [ ] 역방향 화살표가 전체의 20% 이하인가?

### 추상화
- [ ] 한 다이어그램 안에 C4 레벨이 혼합되지 않았는가?
- [ ] 인프라 요소와 비즈니스 로직이 같은 다이어그램에 없는가?

### 선
- [ ] 모든 선의 스타일(실선/점선/굵은선)에 의미가 부여되었는가?
- [ ] 선 라벨이 3단어 이내인가?

### 색상
- [ ] 사용 색상 4개 이하인가?
- [ ] 같은 역할의 도형이 같은 색인가?

### 범례
- [ ] 범례가 존재하는가?
- [ ] 범례에 모든 선 스타일이 설명되었는가?
- [ ] 범례에 모든 색상이 설명되었는가?
- [ ] 축약어가 있으면 범례에 포함되었는가?

### 설명
- [ ] 번호 매기기 또는 흐름 설명이 존재하는가?
- [ ] 도형 안의 축약어가 본문에서 설명되었는가?

---

## 권장 체크 (위반 시 경고, 강제 아님)

- [ ] 그리드 정렬이 시각적으로 깔끔한가?
- [ ] 설명 없이 70% 이상 이해 가능한가?
- [ ] 가장 중요한 흐름이 시각적으로 강조되었는가?
- [ ] 다이어그램 제목이 내용을 잘 설명하는가?

---
