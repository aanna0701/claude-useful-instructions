# Google Style Refactor — Python Agent

Rewrites a batch of Python files to conform to the Google Python Style Guide.

## Required reading

Before editing any file:

1. `rules/google-style-python.md` — project rule checklist
2. The file being modified (in full)
3. `pyproject.toml` — to respect project-level `line-length`, `target-version`, and any `[tool.ruff]` overrides

## Input

A list of Python file paths passed by the orchestrator.

## Steps per file

1. **Verify the mechanical pass ran** — `ruff check <file>` should return no fixable violations, and `ruff format --check <file>` should exit 0. If not, run `ruff check --fix --unsafe-fixes <file>` then `ruff format <file>` first.
2. **Imports**:
   - Enforce the 3-group structure (stdlib, third-party, local), alphabetized, one import per line.
   - Remove `from x import *`. Replace with explicit imports.
   - Prefer absolute imports over relative.
3. **Naming pass**:
   - Classes / exceptions / type vars → `PascalCase`
   - Functions / methods / variables → `snake_case`
   - Module-level constants → `SCREAMING_SNAKE_CASE`
   - "Private" members → `_leading_underscore`
   - Dunder attributes stay dunder.
4. **Type hints** — add to all public function signatures (parameters + return). Use PEP 604 (`X | Y`) style when project targets 3.10+. Add `from __future__ import annotations` at the top of the file if not already present and project is on 3.9.
5. **Docstrings** — Google style. Every public module, class, and function gets one:
   - One-sentence imperative summary, period-terminated.
   - Sections: `Args:`, `Returns:` (or `Yields:`), `Raises:`, `Attributes:` with indented body.
   - Convert reST / NumPy style docstrings to Google style.
6. **Language features**:
   - `.format()` / `%`-formatting → f-strings
   - Mutable default args → `None` + body construction
   - Bare `except:` → specific exception types
   - `eval` / `exec` → flag for review (never rewrite silently)
7. **Forbidden** — remove `from x import *`; flag module-level side effects for review.
8. **Write** the file back.

## When to bail out

- Public API rename (function/class exported via `__all__` or imported by other modules in the repo): **do not rename**. Add `# TODO(google-style): public API rename` and report.
- Test files with intentionally bad examples (style-test fixtures): skip, detect via `conftest.py` or path `tests/fixtures/style/**`.
- Generated code (`# generated by`, protobuf `*_pb2.py`, etc.): skip.

## Output per file

```
<path>: [ok | flagged | skipped]
  changes: <imports N, naming M, types K, docstrings J, other L>
  flags: [list of issues requiring human review]
```

## Verify

After the batch completes, run:

```bash
ruff check <batch>
ruff format --check <batch>
```

Both must exit 0. If mypy is configured for the project, also run `mypy <batch>` and report but do not gate on it — type errors may be pre-existing.
