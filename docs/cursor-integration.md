# Cursor / Antigravity Integration Guide

> **Doc type**: How-to | **Audience**: Developers using the Claude-Codex collab workflow

The Cursor/Antigravity integration adds pipeline orchestration (`/collab-workflow`), **structure scaffolding**, and **codebase verification** to the collab workflow.

---

## Why Cursor/Antigravity?

IDE AI fills three gaps in the Claude Code + Codex workflow:

| Gap | IDE AI Feature | How It Helps |
|-----|---------------|--------------|
| Multi-file structure creation | **Composer** (Cmd+I) | Scaffolds directories, types, and stubs across many files at once |
| Full codebase context | **Codebase search** | Verifies implementation against the entire project, not just changed files |
| Pipeline orchestration | **`/collab-workflow`** | Coordinates ClaudeвҶ’IDEвҶ’CodexвҶ’IDEвҶ’Claude with human gates |

**Cursor/Antigravity is optional.** All workflows work without it. The commands gracefully skip or redirect when IDE AI isn't in use.

---

## Quick Start

1. **Install** the collab bundle (Cursor rules + **slash command** for the full pipeline):

   ```bash
   ./install.sh --collab /path/to/project
   ```

   This writes `.cursor/commands/collab-workflow.md` so **`/collab-workflow`** shows up in CursorвҖҷs `/` menu; `.cursor/rules/collab-pipeline.mdc` and `.agent/workflows/collab-pipeline.md` are **assembled from one source** (`templates/collab-pipeline-body.md` in this repo) so the step-by-step pipeline never drifts between tools.

2. **Full pipeline** (recommended) вҖ” in Cursor chat:

   ```
   /collab-workflow "Add payment module"
   ```

   Flow: Claude plans вҶ’ IDE scaffolds вҶ’ Codex implements вҶ’ IDE verifies вҶ’ Claude reviews.

3. **Or** use individual steps: `/work-plan`, `/work-scaffold FEAT-001`, `bash codex-run.sh FEAT-001`, `/work-review FEAT-001`. For AUDIT items use `/work-verify` (and optional `--ingest`).

## Commands

### `/work-scaffold`

Generates a Cursor/Antigravity Composer prompt from work item contracts.

```
/work-scaffold FEAT-001              # New feature scaffold
/work-scaffold REFAC-002             # Refactoring migration map
/work-scaffold FEAT-001 REFAC-002    # Multiple items
/work-scaffold AUDIT-003             # вҶ’ prints "skip, use /work-verify"
```

**What it does**:
1. Reads `brief.md` and `contract.md` from the work item
2. Auto-detects type from ID prefix (FEAT/REFAC/AUDIT)
3. Fills the type-specific template with contract data
4. Generates `.cursor/rules/*.mdc` in the worktree
5. Updates `status.md` вҶ’ `scaffolded`
6. Prints the prompt for clipboard copy

### `/work-verify`

Generates a Cursor/Antigravity codebase verification prompt.

```
/work-verify FEAT-001               # Implementation verification
/work-verify REFAC-002              # Regression verification
/work-verify AUDIT-003              # Standalone codebase audit
```

**What it does**:
1. Reads `status.md`, `brief.md`, and `contract.md`
2. Auto-detects type from ID prefix
3. Fills the type-specific verification template
4. Prints the prompt for clipboard copy

## Cursor/Antigravity Rules

`.cursor/rules/*.mdc` files are glob-based rules that Cursor/Antigravity applies automatically:

### Project-level (installed by `./install.sh --collab`)

| File | Trigger | Purpose |
|------|---------|---------|
| `collab-pipeline.mdc` | Editing `work/**`, `AGENTS.md`, `CLAUDE.md` | Orchestrates the full planвҶ’scaffoldвҶ’implementвҶ’verifyвҶ’review pipeline with human gates |

### Per-work-item (generated by `/work-scaffold`)

| File | Trigger | Purpose |
|------|---------|---------|
| `{SLUG}-guard.mdc` | Editing files matching allowed modifications | Shows contract boundaries, interfaces, invariants |
| `{SLUG}-forbidden.mdc` | Opening files in forbidden zones | Warns that the file is outside contract scope |

**How it works**: Cursor/Antigravity reads `.cursor/rules/*.mdc` files and matches the `globs` frontmatter against the file being edited. When a match is found, the rule content is injected into the AI's context automatically вҖ” no manual prompt copy needed.

Example generated `FEAT-001-guard.mdc`:
```
---
description: "FEAT-001 contract boundary enforcement вҖ” Add JWT authentication"
globs: ["src/auth/**", "tests/auth/**"]
---

# FEAT-001 вҖ” Contract boundaries
## Allowed Modifications
- src/auth/
- tests/auth/

## Forbidden Zones вҖ” DO NOT MODIFY
- src/core/database.py вҖ” shared infrastructure
- src/models/base.py вҖ” base model definitions
...
```

Cursor/Antigravity reads these files and matches the `globs` frontmatter against the file being edited вҖ” no manual prompt copy needed.

## Pipeline Rule

The `collab-pipeline.mdc` rule describes the full collab workflow inside Cursor/Antigravity. The IDE AI acts as **orchestrator**: it runs scaffold and verify itself, and **delegates** plan, implement, and review to `claude -p` and `codex-run.sh` via the terminal (see the rule body).

### Entry point

```
/collab-workflow JWT мқёмҰқ лҜёл“ӨмӣЁм–ҙ м¶”к°Җн•ҙмӨҳ
```

Or just describe the task naturally вҖ” the rule activates when editing `work/**` files.

### How it works

1. User types `/collab-workflow {instruction}` or requests a feature/fix/refactor/audit
2. AI executes the 6-step pipeline, stopping after each step for confirmation
3. User confirms with "г…Үг…Ӣ", "ok", "м§„н–ү", or "next" to proceed

### Pipeline steps

| Step | Action | Executor | Method |
|------|--------|----------|--------|
| 1. Plan | Generate work item specs | Claude | `claude -p` with work-plan command |
| 2. Scaffold | Create dirs + type stubs | Cursor/Antigravity | Direct file creation |
| 3. Implement | Write code per contract | Codex | `codex-run.sh` or `codex exec` |
| 4. Verify | Check against contract | Cursor/Antigravity | Codebase search |
| 5. Review | Merge/revise decision | Claude | `claude -p` with work-review command |
| 6. Revise | Fix MUST-fix items | CodexвҶ’CursorвҶ’Claude | Re-run steps 3вҶ’4вҶ’5 |

### Revise limits

- Maximum 3 revision rounds
- Same MUST-fix unresolved 2 consecutive rounds: stops and requests manual intervention
- AUDIT items skip Steps 2-3 and go directly to verification

### Installation

```bash
./install.sh --collab /path/to/project
```

Installs to 3 paths for cross-tool compatibility:

| Path | Tool | Format |
|------|------|--------|
| `.cursor/rules/collab-pipeline.mdc` | Cursor / Background Agent | MDC with globs frontmatter |
| `.agent/workflows/collab-pipeline.md` | Antigravity / agent harness | Markdown + minimal YAML frontmatter |
| `AGENTS.md` (pipeline section) | All tools | Cross-tool standard |

**Source of truth:** edit `templates/collab-pipeline-body.md` in **claude-useful-instructions**; `install.sh --collab` prepends the correct frontmatter for each path (no duplicate pipeline text in the repo).

## State Machine

The Cursor/Antigravity integration adds optional states and transitions:

```
planned вҶ’ [scaffolded] вҶ’ implementing вҶ’ ready-for-review вҶ’ reviewing вҶ’ merged
             вҶ‘ optional
             (.cursor/rules/ generated)

planned вҶ’ auditing вҶ’ audited   вҶҗ AUDIT only (/work-verify --ingest)
```

- `scaffolded`: Set by `/work-scaffold`. Generates `.cursor/rules/*.mdc`. `codex-run.sh` accepts both `planned` and `scaffolded`.
- `auditing`: Set by `/work-verify` for AUDIT items. No implementation phase.
- `audited`: Set by `/work-verify --ingest` for AUDIT items on PASS verdict.

## Verify Result Ingestion

After running `/work-verify AUDIT-NNN` and getting Cursor/Antigravity's output, ingest the results:

```
/work-verify AUDIT-001 --ingest
# вҶ’ Paste Cursor/Antigravity output вҶ’ auto-parsed вҶ’ verdict + routing
```

The `--ingest` mode:
1. Parses structured findings (audit table format)
2. Saves to `work/items/{ID}-*/verify-result.md`
3. Auto-determines verdict: PASS / PASS_WITH_WARNINGS / FAIL
4. Routes: `audited` status with options to create issues or plan fixes

## Cross-Tool Compatibility

The pipeline rule is installed to 3 paths so it works across tools:

| Tool | Rules Path | Pipeline File |
|------|-----------|---------------|
| Cursor (interactive) | `.cursor/rules/*.mdc` | `collab-pipeline.mdc` (glob-triggered) |
| Cursor Background Agent | `.cursor/rules/*.mdc` | Same file (read from repo clone) |
| Antigravity (Google) | `.agent/workflows/*.md` | `collab-pipeline.md` |
| Codex / Claude Code | `AGENTS.md` | Pipeline section (pointer to rule files) |

### Skills vs rules (single tree)

Bundled **skills** (`SKILL.md` trees) live only under **`.claude/skills/<name>/`**. Cursor does not get a duplicate under `.cursor/skills/` from this installer; use **`.cursor/rules/*.mdc`** (e.g. `collab-pipeline.mdc`) and `AGENTS.md` for IDE-side workflow context. If you rely on CursorвҖҷs skill picker, point it at `.claude/skills` manually or adopt CursorвҖҷs native `.cursor/skills/` layout yourselfвҖ”this repo standardizes on one path for Claude Code.

### Without Cursor or Antigravity

Every workflow works without AI IDE integration:

| Phase | With AI IDE | Without |
|-------|-------------|---------|
| Pipeline | `collab-pipeline.mdc` auto-orchestrates | Manual `/work-plan` вҶ’ `codex-run.sh` вҶ’ `/work-review` |
| Scaffold | Composer scaffolds structure | Codex creates files from scratch |
| Rules | `.cursor/rules/*.mdc` auto-enforces contract | Codex relies on AGENTS.md boundaries |
| Audit | `/work-verify` for AUDIT-type items | Manual review |
| Diff review | Side-by-Side diff | `git diff` in terminal |
