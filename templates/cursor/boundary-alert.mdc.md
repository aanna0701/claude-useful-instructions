# Boundary Alert Rule Template

Forbidden Zone files trigger a warning rule.
When Cursor/Antigravity opens a file matching these globs, it displays a boundary violation warning.

Generated by `/work-scaffold` from contract Forbidden Zones.

---

## Template

```
---
description: "{{item_id}} — FORBIDDEN: this file is outside contract boundaries"
globs: [{{#each forbidden_globs}}"{{this}}"{{#unless @last}}, {{/unless}}{{/each}}]
---

DO NOT modify this file.

This file is in the Forbidden Zone for work item {{item_id}}.

If this file needs changes:
1. Record the need in status.md under "## Blockers"
2. Do NOT implement the change yourself
3. The contract owner (Claude) will handle it separately

Objective: {{objective}}
Allowed files: see .cursor/rules/{{slug}}-guard.mdc
```

## Variable Sources

| Variable | Source | Derivation |
|----------|--------|------------|
| `item_id` | work item ID | e.g., `FEAT-001` |
| `objective` | `brief.md` > `## Objective` | Direct extraction |
| `slug` | work item directory name | e.g., `FEAT-001-jwt-auth` |
| `forbidden_globs` | `contract.md` > Forbidden Zones | Path → glob pattern |

## Glob Derivation Rules

Same as `contract-guard.mdc.md`:

| Path Pattern | Glob Output |
|-------------|-------------|
| `src/core/` (directory) | `src/core/**` |
| `src/models/base.py` (file) | `src/models/base.py` |
