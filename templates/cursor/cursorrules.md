# .cursorrules Generation Template

Use this template to generate a `.cursorrules` file in the worktree root.
Generated by `/work-scaffold` from contract + AGENTS.md.

---

## Template

```
# Project Rules for Cursor AI

## Work Item Context

Active work item: {{item_id}}
Objective: {{objective}}
Type: {{item_type}}

## Boundaries — STRICTLY ENFORCED

### Never Modify These Files

{{#each forbidden_zones}}
- {{path}}{{#if reason}} — {{reason}}{{/if}}
{{/each}}

### Allowed Modifications Only

You may ONLY modify files listed here:
{{#each allowed_modifications}}
- {{this}}
{{/each}}

## Coding Conventions

- Language: {{primary_language}}
- Keep code and comments in English
- Follow existing project formatting (indentation, line length, import order)
- Prefer immutable data patterns — create new objects instead of mutating
- Functions should be < 50 lines, files < 800 lines
- Handle errors explicitly — never silently swallow errors
- No hardcoded secrets (API keys, passwords, tokens)

## Tech Stack

{{#each dependencies}}
- {{this}}
{{/each}}

## Non-Negotiables

- Never implement on the parent/main branch
- Never create sub-branches from feature branches
- Never modify brief.md, contract.md, or checklist.md (read-only)
- Update status.md when making progress
- Record documentation changes needed in status.md (never modify docs directly)
```

## Variable Sources

| Variable | Source File | Section |
|----------|------------|---------|
| `item_id` | work item ID | e.g., `FEAT-001` |
| `objective` | `brief.md` | `## Objective` |
| `item_type` | ID prefix | `FEAT` / `REFAC` / `AUDIT` |
| `forbidden_zones` | `contract.md` | `## Boundaries > ### Forbidden Zones` |
| `allowed_modifications` | `contract.md` | `## Boundaries > ### Allowed Modifications` |
| `primary_language` | inferred | From file extensions in Allowed Modifications |
| `dependencies` | `brief.md` | `## Dependencies` |
