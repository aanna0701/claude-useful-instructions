# Contract Guard Rule Template

Use this template to generate `.cursor/rules/{SLUG}-guard.mdc` in the worktree.
Generated by `/work-scaffold` from contract boundaries.

---

## Template

```
---
description: "{{item_id}} contract boundary enforcement — {{objective}}"
globs: [{{#each allowed_globs}}"{{this}}"{{#unless @last}}, {{/unless}}{{/each}}]
---

# {{item_id}} — Contract boundaries

## Objective
{{objective}}

## Allowed Modifications
{{#each allowed_modifications}}
- {{this}}
{{/each}}

## Forbidden Zones — DO NOT MODIFY
{{#each forbidden_zones}}
- {{path}}{{#if reason}} — {{reason}}{{/if}}
{{/each}}

## Interface Contract
{{#each interfaces}}
- {{name}} ({{type}}) in {{owner}}: {{spec}}
{{/each}}

## Invariants
{{#each invariants}}
- {{this}}
{{/each}}

When editing files matching the globs above:
- Stay within Allowed Modifications only
- Warn before touching anything in Forbidden Zones
- Implement interfaces exactly as specified in the contract
- Preserve all invariants
```

## Variable Sources

| Variable | Source | Derivation |
|----------|--------|------------|
| `item_id` | work item ID | e.g., `FEAT-001` |
| `objective` | `brief.md` > `## Objective` | Direct extraction |
| `allowed_globs` | `contract.md` > Allowed Modifications | Path → glob pattern (e.g., `src/auth/` → `src/auth/**`) |
| `allowed_modifications` | `contract.md` > Allowed Modifications | Direct extraction |
| `forbidden_zones` | `contract.md` > Forbidden Zones | Direct extraction |
| `interfaces` | `contract.md` > Interfaces | Table row extraction |
| `invariants` | `contract.md` > Invariants | List extraction |

## Glob Derivation Rules

Convert `Allowed Modifications` paths to glob patterns:

| Path Pattern | Glob Output |
|-------------|-------------|
| `src/auth/` (directory) | `src/auth/**` |
| `src/auth/token.py` (file) | `src/auth/token.py` |
| `src/auth/*.py` (already glob) | `src/auth/*.py` |
| `tests/` (directory) | `tests/**` |
